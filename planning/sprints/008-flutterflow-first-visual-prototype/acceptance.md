# Sprint 008 Acceptance Criteria - FlutterFlow First Visual Prototype

Sprint 008 is complete only when all criteria below are satisfied.

## Scope Acceptance

- Sprint 008 folder exists at `planning/sprints/008-flutterflow-first-visual-prototype/`.
- The sprint includes:
  - `requirements.md`
  - `blueprint.md`
  - `acceptance.md`
  - `handoff-prompt.md`
- `planning/STATE.md` identifies Sprint 008 as current.
- Relevant planning/docs files are updated only where needed.
- No local runtime app code is changed.
- No FlutterFlow screens are created inside this repo.
- No generated FlutterFlow exports are added.
- No production mobile release files are created.
- No files are staged, committed, or pushed.

## Screen Acceptance

The sprint defines these six screens:

- Splash / Loading
- Welcome / Intro
- Sign In / Demo Access
- Home Dashboard
- Chat Shell
- Error / Offline

## Visual Identity Acceptance

The sprint documents screenshot-derived visual requirements:

- Pale pink background.
- Hot pink primary CTA.
- Soft rounded cards.
- Mobile adaptation of left-side quick action/navigation concept.
- Large central microphone action.
- Executive assistant feel.
- `Strategic Advisor Mode`.
- `CEO Briefing`.
- Finance quick action.

## Theme Acceptance

- `trina-barbie-v1` is documented as the first visual prototype direction.
- Theme tokens are documented.
- Theme configurability remains documented.
- Barbie-inspired styling is not treated as permanent hard-coded identity.

## Data/API Acceptance

- Mock data is documented.
- API stubs are documented.
- `mockStartDemoSession` is documented.
- `mockSendAssistantMessage` is documented.
- `mockMicListeningState` is documented.
- Live Firebase wiring is out of scope.
- Live Gmail wiring is out of scope.
- Live backend API wiring is out of scope.
- Live auth is out of scope.
- Live model calls are out of scope.
- Secrets/API keys are forbidden.

## Validation Acceptance

- `docs/VALIDATION.md` includes Sprint 008 docs-only validation checks.
- Validation commands are run and reported.
- Final report confirms no runtime code changed.
- Final report confirms no generated exports or production mobile release files were created.
- Final report confirms no staging, commit, or push occurred.

## Completion Standard

Sprint 008 is complete only when the repo contains a clear docs/planning package that can guide the first FlutterFlow visual prototype build without requiring the Builder to infer screen list, visual identity, theme, mock data, API stubs, or backend boundaries.
